l'essenstiel de vscode:
	intro:
		vscode-cli:
			code -v | code -h
			code
			code .
			code -n 
			code -d Program.cs Program2.cs	 //-->ouvre un editeur de difference des deux fichiers
			
			code -g Program.cs:9	//-->pour ouvrir le fichier susnomme a la ligne 9

		decouvrir l'interface:
			on a la barre d'activite tout a gauche, qui est donc suivi de la sidebar, et enfin la zone d'edition
				en bas, on a eventuellement le panel composé du terminal, de la console de debug, etc
					et en haut, on a le menu et eventuellement la palette de commande

		les fichiers et les fenetres d'edition:
			ouvrir des onglets:
				mode de previsualisation: 1 clic sur le fichier donné
					mode d'edition: double sur le fichier donné
				CTRL+SHIFT+TAB pour passer d'un onglet à un autre

			ouvrir des fenetres-onglets d'edition:
				ouvrir sur le cote (ctrl+enter) == fractionner l'editeur (ctrl+\)
					ou encore (ctrl+double_click_sur_le_fichier_donné)
						fermer les fenetres d'edition: CTRL+W

				switcher entre les fenetres d'edition:
					CTRL+1 / CTRL+2 / CTRL+3
					ou ctrl+k,ctrl+right_arrow / ctrl+k,ctrl+left_arrow

		les espaces de travail ou workspace

		les preferences:
			les raccourcis claviers
			les keymaps
			les snippets
			les themes
			les parametres:
				parametres par defaut
					puis parametres utilisateur
						puis parametres du workspace
							puis parametre du dossier de travail

	Travailler avec le code:
		la palette de commandes:
			CTRL+P -> to access all file within workspace
			CTRL+P,? 	-> to list all global commands
			CTRL+P,> 	-> to show and run all other palette commands
			CTRL+P,#	-> to see all available symbol in a workspace
			CTRL+P,@:	-> to see all available symbol in a specific file by category

		IntelliSense (code.visualstudio.com/docs/editor/intellisense):
			CTRL+SPACE: pour forcer l'affichage d'intellisense
				CTRL+SPACE encore, une fois l'autocompletion active pour afficher/caher l'aide contextuelle

		formatage:
			format all the document: SHIFT+ALT+F 	format the selection: CTRL+K,CTRL+F
				activate/deactivate editor.formatOnSave, formatOnType, formatOnPaste in preferences
		
		linter, codelens:
			linter:
				CTRL+SHIFT+M  	-> pour afficher la console de blem-pro
				CTRL+. 	-> afficher les correctifs

			codelens: 
				affiche a l'interieur du code des infos supplementaires
					regexpreviewer, gitlens par exemple utilise l'api de codelens

				F12: Aller au fichier de la definition
				ALT+F12: Previsualison de la definition

		connaitre les raccourcis claviers propres a vscode: CTRL+K,CTRL+R

		selections multiples:
			multiCursor (editor.multiCursorModifier: ALT)
				multiselection peu importe en ligne ou en colonne, par lettre ou par mot entier

			etendre sa selection vers le haut et/ou le bas: SHIFT+ALT+right_arrow/left_arrow 
				
			COPIER-COLLER:
				SHIFT+ALT+top_arrow/bottom_arrow
				ou bien, il suffit de placer le curseur n'importe sur la ligne desirée puis CTRL+C,CTRL+V

			selectionner en colonne: 
				SHIFT+ALT+(glisser_de_la_souris)
				
				sinon pour donc etendre la selection en colonne d'un mot deja selectionné:
					CTRL+SHIFT+ALT+top_arrow/bottom_arrow
			
			selection d'occurence: 
				CTRL+Fn+F2
				si un premier mot est deja selectionné: CTRL+SHIFT+L

		modifier l'affichage:
			codefolding integré: c-a-d la possibilité de plier/deplier du code
				CTRL+K,CTRL+0 pour tout plier
				CTRL+K,CTRL+J pour tout deplier
			y'a aussi la possibilité de definir des REGIONS dans le code selon le langage, 
				voir la doc de vscode

			go to VIEW:
				fullscreen: F11
				zenmode: CTRL+K,Z pour l'activer et la meme chose/Escape+Escape pour le desactiver
		
		effectuer des recherches:
			recherche global sur le projet en entier: CTRL+SHIFT+F
			recherche dans le fichier courant: CTRL+F
				ENTER ou F3 pour avancer dans la recherche vers les occurences suivantes
			remplacement dans ke fichier courant: CTRL+H

	Aller plus loin:
		les tasks runner de vscode (npm, grunt, gulp, tsc, c#....):
			si le task runner a son propre fichier de config, vscode y extrait directement les taches
				comme avec le tsconfig pour typescript...
			
			sinon en definissant le dossier de travail comme un espace de travail vscode, 
				vscode y cree un fichier tasks.json dans le dossier .vscode (.vscode/task.json)
					tel est le cas pour un projet c#/dotnet par exemple

		git for vscode:
			some extensions: vscode-github, gitlens

		le debogage (.vscode/launch.json utilise les debuggers integrés aux extensions vscode)

		emmet (docs.emmet.io/cheat-sheet)
